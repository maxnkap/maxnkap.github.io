CNC Machined Map Project

Project Background: After taking a long road trip, I wanted to have a memento of the trip I took. Despite having many cell-phone photos, I wanted something a little more permanent. With a few weeks before I was set to start my internship at Liquid Robotics I decided to see if I could machine a map of my trip.
This project uses a pair of python scripts written to extract your personal gps tracking data recorded from a cellphone running android, and generate a personalized, physical map of your trip. This data is collected passively if you have the access to my location setting turned on on your android device, and can be accessed at: https://maps.google.com/locationhistory/b/0 Because this data is collected passively, you can make these maps of any interesting trip you have made at any time in the past.

Overview of the Map Generation Process
  • Clone code from Github.
  • Download your personal tracking data in KML from Google.
  • Preprocess the data, removing ‘gx:’ from kml tags in your preferred text editor.
  • Use MapUnique.py to generate a latitude longitude list in csv that can be plotted.
  • Write a csv file of any stops you want to display.
  • Use USMapFromLat.py to generate your map in PostScript format.
  • Use pstoedit to convert your PostScript map to a dxf file.
  • Import your map in dxf to your preferred 3D CAD package.
  • Clean up your dxf and extrude to a solid.
  • Use your preferred CAM package to prepare gcode.
  • Machine the map.

Two main data locations to get data from. If you want a short period of time, go to https://maps.google.com/locationhistory/b/0 and use the dropdown menu to select the time frame you want to get data from.

Note: You can also get all of your historical data from https://www.google.com/settings/takeout but note that this is likely to be at least a million data points. Note also that this data is in JSON form which is much easier to work with, but requires a different set of scripts which can be found here instead.

Unfortunately due to limitations in the Python code I wrote, it is not possible to fully strip the KML tags when extracting latitude, longitude data, so until further development is done, a find/replace of all 'gx:' tags is necessary. It is then possible to extract each latitude longitude pair from the KML that Google provides, and plot each unique data point using Matplotlib. This map can be exported to a .ps file format, which is able to be converted to a dxf and then imported to Solidworks.

Once imported into Solidworks, some extraneous lines need to be removed, but it is fairly straightforward to clean up the file. This dxf can be used to define a trace path for HSMWorks, and conveniently the outlines of the map are a separate layer of the dxf from the plotted data points from my trip.

3 machining operations later (plus a coat of enamel paint to clean things up) the final map is done.
